Цей гайд призначений для швидкого розгортання проєкту на будь-якому комп'ютері, де встановлені Git та Docker Desktop. Завдяки наявності попередньо налаштованої бази даних, застосунок готовий до роботи одразу після запуску контейнерів.

Крок 1: Клонування репозиторію
Відкриваємо термінал (або PowerShell) у будь-якій зручній папці на комп'ютері та завантажуємо код:

Bash
git clone https://github.com/ТВІЙ_ЛОГІН/ТВІЙ_РЕПОЗИТОРІЙ.git
Після завантаження переходимо в папку з проєктом:

Bash
cd ТВІЙ_РЕПОЗИТОРІЙ
Крок 2: Запуск контейнерів Docker
Переконуємося, що програма Docker Desktop увімкнена і горить зелений індикатор (Engine running).
В терміналі проєкту запускаємо збірку та старт нашого сервера:

Bash
docker compose up --build
Docker автоматично завантажить потрібну версію Python, встановить залежності (Django, Pillow) та підніме сервер. Оскільки файл бази даних (db.sqlite3) та папка з картинками (media/) вже знаходяться у папці проєкту, Docker через механізм volumes миттєво підтягне їх усередину контейнера.

Крок 3: Перевірка роботи сайту (Товари вже на місці!)
Тепер усе повністю готово. Відкриваємо браузер:

Головна сторінка магазину: Перейти за адресою http://127.0.0.1:8000/.

Сайт одразу відображатиметься у повному дизайні, з наявними категоріями та реальними товарами (Сукня, Шкіряна сумка тощо) та завантаженими фотографіями.

Крок 4: Керування через Адмін-панель (Опціонально)
Якщо викладач захоче зайти в панель адміністратора або створити власний профіль:

Вхід в наявну адмінку: Перейти за адресою http://127.0.0.1:8000/admin/ та ввести логін і пароль суперкористувача, які були створені під час локальної розробки.

Створення нового (додаткового) адміна: Якщо потрібно створити новий обліковий запис адміністратора, відкриваємо друге вікно терміналу у тій же папці проєкту і запускаємо команду:

Bash
docker compose exec web python manage.py createsuperuser
Термінал попросить ввести новий логін, пошту та пароль (символи пароля вводяться наосліп).

Крок 5: Запуск тестів (Для перевірки стабільності в CI/CD)
Щоб продемонструвати викладачу, що тести проходять успішно прямо всередині працюючого Docker-контейнера, у сусідньому вікні терміналу запускаємо команду:

Bash
docker compose exec web python manage.py test
🛑 Як правильно зупинити роботу?
Коли роботу завершено, у першому терміналі (де відображаються логи сервера) просто натискаємо комбінацію клавіш Ctrl + C. Сервер зупиниться.

Якщо необхідно повністю погасити, підчистити контейнери та звільнити оперативну пам'ять, у сусідньому терміналі пишемо:

Bash
docker compose down
